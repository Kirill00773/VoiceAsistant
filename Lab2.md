# Лабораторная работа №2  
## Выбор технологий для разработки локального голосового ассистента

**Проект:** Локальный голосовой ассистент для студентов лаборатории  
**Тип работы:** Теоретическое обоснование выбранного стека  

---

## 1. Язык разработки

В качестве основного языка выбран **Python**.

Выбор обусловлен тем, что Python является стандартом де-факто в задачах, связанных с машинным обучением и обработкой естественного языка. Для него уже существуют готовые библиотеки для работы с LLM, векторными базами данных, распознаванием и синтезом речи.  

Кроме того, Python проще поддерживать и дорабатывать в рамках учебного проекта.

В проекте Python будет использоваться для:
- оркестрации работы всех модулей (LLM, STT, TTS),
- реализации логики RAG (поиск по базе знаний),
- обработки аудиопотоков в реальном времени,
- запуска локального сервера.

---

## 2. Локальная языковая модель и среда запуска

В качестве среды запуска выбран Ollama, а в качестве модели — Llama 3 8B (квантованная версия).

Ollama позволяет запускать LLM как локальный сервис, автоматически управляя видеопамятью (VRAM). Модель Llama 3 8B выбрана как оптимальное решение по соотношению «качество ответов / требование к ресурсам» для современного ПК. Квантование (4-bit) позволяет модели работать на 8-16 ГБ оперативной памяти.

Модель будет использоваться для:
- генерации ответов на естественном языке,
- обобщения найденной информации из базы знаний,
- поддержания контекста диалога.

---

## 3. Архитектура RAG

Для работы с базой знаний выбрана архитектура **Retrieval-Augmented Generation (RAG)**.

LLM сама по себе не обладает знаниями о документах лаборатории. Поэтому используется подход, при котором перед генерацией ответа сначала выполняется поиск релевантной информации в базе знаний.

Общий принцип работы:

1. Документы (инструкции, FAQ) разбиваются на смысловые фрагменты (чанки).
2. Фрагменты преобразуются в векторное представление (эмбеддинги).
3. При запросе пользователя ищутся наиболее близкие по смыслу фрагменты.  
4. Найденный контекст передаётся в модель.  
5. Модель формирует ответ с опорой на этот контекст.  

Такой подход позволяет снизить вероятность генерации недостоверной информации и сделать ответы более точными.

## 4. Векторная база данных

Для хранения эмбеддингов выбрана база данных ChromaDB.

В отличие от простых библиотек поиска, ChromaDB является полноценной векторной базой данных, способной хранить не только векторы, но и метаданные, а также сам текст документов. Она работает локально (без сервера), сохраняя данные в файлы, и имеет нативную интеграцию с фреймворком LangChain.

В рамках проекта ChromaDB будет использоваться для:
- индексации базы знаний лаборатории,
- семантического поиска информации по запросу студента.

---

## 5. Распознавание речи (STT)

Для преобразования речи в текст выбрана библиотека Faster-Whisper.

Это оптимизированная реализация модели Whisper от OpenAI (на базе CTranslate2). Она работает в 4–5 раз быстрее оригинальной версии, потребляя меньше памяти, что критически важно для снижения задержки ответа (latency) в диалоговой системе.

Faster-Whisper будет использоваться для:
- мгновенной транскрибации речи с микрофона,
- передачи распознанного текста в RAG-конвейер.

## 6. Синтез речи (TTS)

Для озвучивания ответов выбрана нейросеть Silero TTS.

На текущий момент это одно из лучших решений для синтеза русской речи в режиме оффлайн. Она отличается высокой скоростью работы (ответ генерируется быстрее, чем произносится) и компактным размером, что позволяет запускать её даже на процессоре (CPU).

В проекте Silero TTS будет использоваться для:
- генерации аудиопотока из текстового ответа LLM,
- обеспечения естественного звучания голоса ассистента.

---

## 7. Организация взаимодействия компонентов

Для связывания логики RAG и интерфейса выбран фреймворк LangChain в связке с FastAPI.

LangChain используется для построения цепочек обработки данных (Retriever -> Prompt -> LLM), а FastAPI — для создания локального API сервера, к которому будет обращаться микрофон и динамики.

Такая связка обеспечивает модульность системы и упрощает дальнейшее расширение функционала.
